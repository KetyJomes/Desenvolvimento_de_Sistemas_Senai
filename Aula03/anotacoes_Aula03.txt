Aula 03: Introdução API´s

Usando Nodjs

1 - Criar arquivos
- npm init
- npm init -y
- npm i 

2 - Comando personalizado para rodar no terminal:

- package.json
- "scripts":{
    "start": "node teste.js" (agora no terminal você vai usar 
    npm run start para rodar)
}
OBS: Caso eu queira rodar sem personalizar: 
node nome do arquivo Ex: node teste.js

3 - Instalar typescript
- npm i typescript @types/node tsx -D

4 - Gera um arquivo de tsc para o js
- npx tsc --init

- pesquisar tsc config base e entra no git do cara
- copiar os comandos da nossa versão do node:

npm install --save-dev @tsconfig/node22
yarn add --dev @tsconfig/node22 (versão do meu node é o 22)

- Trocar o código que está no tsconfig por esse:

{
  "$schema": "https://www.schemastore.org/tsconfig",
  "_version": "22.0.0",

  "compilerOptions": {
    "lib": ["es2024", "ESNext.Array", "ESNext.Collection", "ESNext.Iterator"],
    "module": "nodenext",
    "target": "es2022",

    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "moduleResolution": "node16"
    "allowImportingTsExtensions": true,
    "noEmit": true,
  }
}

5 - arquivo server.ts

colocar no script do package.json:
"dev": "npx tsx watch src/serve.ts"

rodar no terminal:
npm i express @types/express

6 - Criar arquivo src e dentro dele:
uma pasta routes e dentro dela:
arquivo routes.ts e o arquivo que voce coloca o post, get e put.

POSTMAN

1 - do lado do meu nome workspace clicar em new - http
2 - para ver se esta funcionando colocar do lado do get localhost:8080
(8080 é a porta que esta em nosso arquivo server.ts)
3 - para registrar - localhost:8080/api/é o que esta na routes.ts/é o que esta no outro arquivo post
